StudentTasker — diagram architektury

Aplikacja internetowa PHP w architekturze MVC z bazą PostgreSQL i Dockerem.

Użytkownik / Przeglądarka

Użytkownik korzysta z aplikacji przez przeglądarkę internetową.

↓

public/index.php

Główny punkt wejścia aplikacji, uruchamia sesję i router.

↓

Router

Dopasowuje adres URL i metodę HTTP do odpowiedniego kontrolera.

↓

Controllers

Obsługują żądania użytkownika, sprawdzają sesję, role i uprawnienia.

↓

Services

Logika aplikacji, np. logowanie, sesje i autoryzacja.

Repositories

Komunikacja z bazą danych przez zapytania SQL.

↓

PostgreSQL Database

Tabele, relacje, widoki, funkcje, wyzwalacze i dane testowe.

↓

Views + HTML/CSS/JavaScript

Prezentacja danych, formularze, Fetch API i responsywny interfejs.

Docker uruchamia kontenery aplikacji PHP/Apache oraz bazy PostgreSQL.
